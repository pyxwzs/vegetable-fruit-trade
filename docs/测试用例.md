# 果蔬贸易系统 — 测试用例（毕业论文配套）

本文档既可作**联调与验收清单**，也可直接支撑论文**「系统测试」**章节：将下表用例归纳为「功能测试」「接口测试」「界面冒烟测试」等小节，并在正文中给出**测试环境、步骤、预期与实际结果**（建议配截图或关键响应字段说明）。

**与论文章节的对应建议：**

| 论文「系统测试」常见小节 | 本文对应章节 |
|--------------------------|----------------|
| 测试目的与范围 | 第 0～1 节环境说明 |
| 功能测试（按模块） | 第 2～10 节用例表 |
| 界面/易用性（可选） | 第 11 节 |
| 测试结论与缺陷分析 | 根据执行结果自撰；可引用第 12 节作为后续工作 |

**约定**：接口基准地址为 `http://<主机>:8080/api`（与 `application.yml` 中 `context-path` 一致）。除明确标注外，请求头需带：

```http
Authorization: Bearer <access_token>
Content-Type: application/json
```

响应体统一为 `ApiResponse`：`code === 200` 表示成功（以实际后端约定为准）。

## 1. 测试环境与前置条件

| 项 | 说明 |
|----|------|
| 后端 | 已启动 Spring Boot，端口与 `application.yml` 一致 |

**敏感数据**：勿在用例中写死生产密码；使用测试库与测试账号。

## 2. 认证模块

| 用例编号 | 名称 | 前置条件 | 步骤 | 预期结果 |
|----------|------|----------|------|----------|
| AUTH-01 | 获取认证配置 | 无 | `GET /auth/config` | `200`，body 含 `mfaFeatureEnabled` 等字段 |
| AUTH-02 | 正确密码登录（无 MFA） | 系统/账号均未强制 MFA | `POST /auth/login`，body：`{"username":"...","password":"...","rememberMe":false}` | 返回 `token`（及可选 `refreshToken`），无 `mfaRequired` |
| AUTH-03 | 错误密码 | 同左 | 错误密码 | 业务错误码或 `500` + 明确 `message`，不返回有效 token |
| AUTH-04 | MFA 登录（若开启） | `mfaFeatureEnabled` 且账号开启 MFA | 先 `POST /auth/login` 得到 `mfaRequired`；再按接口文档完成 challenge/verify | 最终获得 `token` |
| AUTH-05 | 刷新令牌 | 持有有效 `refreshToken` | `POST /auth/refresh`，body：`{"refreshToken":"..."}` | 返回新 `token`（及可能的新 `refreshToken`） |
| AUTH-06 | 无效 refresh | 任意 | 伪造 refresh 调用 `/auth/refresh` | 失败，不颁发新 token |
| AUTH-07 | 当前用户 | 已登录 | `GET /auth/me` | 返回当前用户信息与角色等 |
| AUTH-08 | 未携带 Token 访问受保护接口 | 清除 Authorization | `GET /auth/me` 或任意业务 GET | `401` 或后端约定未认证表现 |
| AUTH-09 | 忘记密码发码 | 邮件服务已配置 | `POST /auth/forgot-password`，合法 `email` | 成功响应（具体是否真发信视环境） |
| AUTH-10 | 重置密码 | 已拿到邮箱验证码 | `POST /auth/reset-password` | 可用新密码登录 |
| AUTH-11 | 管理员数据视角（若启用） | 管理员账号 | `GET /auth/data-scope`、`PUT /auth/data-scope` | 读写符合权限设计 |

## 5. 库存核心

| 用例编号 | 名称 | 步骤 | 预期 |
|----------|------|------|------|
| INV-01 | 入库（新批次） | `POST` 入库（商品、仓库、数量、可选批次/生产日期） | 库存增加，`availableQuantity` 与 `quantity` 一致 |
| INV-02 | 入库（已有批次累加） | 同一商品+仓库+批次再次入库 | 数量累加 |
| INV-03 | 出库成功 | 出库数量 ≤ 可用库存 | 扣减成功 |
| INV-04 | 出库失败（不足） | 出库数量大于 `availableQuantity` | 返回业务错误（如「库存不足」），库存不变 |
| INV-05 | 库存列表筛选 | 按商品、仓库、关键字查询 | 结果符合筛选条件 |
| INV-06 | 临期/低库存预警（若接口暴露） | 调用 `InventoryService` 对应查询的 API | 列表与阈值逻辑正确 |

## 12. 自动化测试建议（后端）

- `@SpringBootTest` + `@AutoConfigureMockMvc` 对 `AuthController`、`InventoryController` 做 MockMvc 集成测试；

## 13. 论文中撰写「测试结果」的提示

- 选**覆盖核心业务流程**的用例写入正文（如：登录 → 采购建单 → 审核 → 收货入库 → 销售下单 → 审核 → 发货出库 → 财务或库存查询），不必罗列全部编号。
