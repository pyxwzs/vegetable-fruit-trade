## 1. 总体架构

- **前后端分离**：浏览器访问 Vue 单页应用；通过 HTTP 调用 Spring Boot 提供的 REST API。
- **API 前缀**：后端 `application.yml` 中 `server.servlet.context-path: /api`，故业务路径形如 `/api/auth/login`。
- **开发联调**：前端 `vue.config.js` 将 `devServer` 上的 `/api` 代理到后端 `http://localhost:8080`，由后端再挂载到 `/api`，避免跨域问题。

## 6. 认证与令牌流转（模块协作）

1. 用户在前端提交账号密码 → `POST /api/auth/login`。
2. 若系统与账号启用 MFA → 返回 `mfaRequired` 与 `sessionId`，前端再走 `mfa-challenge` / `mfa-verify` 或合并流程（以当前 `user` 模块实现为准）。
3. 成功后前端保存 `token`（及可选 `refreshToken`）到 `localStorage`；`request.js` 在请求头附加 `Authorization: Bearer <token>`。
4. 后端 `JwtAuthenticationFilter` 校验 access JWT，通过后填充 `SecurityContext`。
5. 访问令牌过期时，前端拦截 401，用 `POST /api/auth/refresh` 换新 token 并重试原请求。

## 8. 小结

- **后端**：Spring Boot + JPA + Security + JWT，按域划分 Controller/Service/Repository。  
- **前端**：Vue 3 生态 + Element Plus，按功能拆分路由、Vuex 与 `api` 封装。  
- **联调**：统一 `/api` 前缀 + 开发代理，保证前后端职责清晰、部署时可独立扩缩。
